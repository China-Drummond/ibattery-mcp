ibattery-mcp

Every Apple battery you own, one MCP call away.

你所有的苹果设备电量，一次 MCP 调用就能看到。

Requires macOS 13 (Ventura) or later, and Homebrew.

需要 macOS 13 (Ventura) 及以上版本，以及 Homebrew。

🔋Install ibattery-mcp

🔋安装 ibattery-mcp

$ brew install China-Drummond/tap/ibattery-mcp

Paste that into a Terminal on this Mac. It pulls in libimobiledevice and pkg-config as dependencies and builds ibattery-mcp from source.

在这台 Mac 的终端里粘贴运行即可。它会一并装上依赖 libimobiledevice 和 pkg-config，并从源码构建 ibattery-mcp。

Then add it to your MCP host's config:

然后把它加到你的 MCP 宿主配置里：

{
  "mcpServers": {
    "ibattery-mcp": { "command": "ibattery-mcp" }
  }
}

One-time setup for Bluetooth devices — start the companion helper app once, and grant it Bluetooth access when asked:

蓝牙设备的一次性设置 —— 启动一次伴生辅助 App，并在弹窗里允许它访问蓝牙：

$ open "$(brew --prefix ibattery-mcp)/libexec/ibattery-ble-helper.app"

One-time setup for iPhone/iPad/Watch — connect the device over USB once and tap "Trust."

iPhone/iPad/Watch 的一次性设置 —— 用数据线连接一次设备，并点击"信任"。

🔧What Does It Do?

🔧它能做什么？

Three tools, callable by any MCP client. get_all_devices_status() is the one a morning-briefing skill calls — everything, in one round trip:

三个工具，任何 MCP 客户端都能调用。晨间简报这类场景直接调 get_all_devices_status()——一次拿到所有设备：

$ tools/call get_all_devices_status
MacBook Pro    mac        87%  charging
iPhone 17      iosDevice  48%
Apple Watch    watch      66%

macOS attributes Bluetooth's privacy check to the responsible process — never an MCP subprocess, since it's forked directly by its host, not launched normally. So a small companion app owns Bluetooth instead, and the stateless server just asks it questions over a socket:

macOS 会把蓝牙隐私检查的责任归到负责进程身上——而 MCP 子进程从来都不是它，因为是被宿主直接 fork 出来的，不是正常启动的。所以蓝牙访问权限交给一个独立的小型伴生 App，无状态的 server 只通过 socket 向它提问：

ibattery-mcp (spawned per request) | | unix socket v ibattery-ble-helper (launched via open, runs continuously)

📦Available Tools

📦可用工具

get_all_devices_status()
Battery/status for every device discoverable from this Mac right now.
现在从这台 Mac 能发现的所有设备的电量/状态。
get_device_battery(query)
Battery status for one device matching a name or type substring, e.g. "iPhone".
查询名字或类型匹配某个关键词的单个设备，比如 "iPhone"。
list_known_devices()
Devices seen so far this session, without triggering a fresh scan.
本次会话里已经看到过的设备，不触发新的扫描。
